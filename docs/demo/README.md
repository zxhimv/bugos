# bugos Synthetic Demo

Status: synthetic demo only

This directory implements REV-004 from `docs/revenue_backlog.md`.

## Purpose

The demo shows how `bugos` can present a prepare-only workflow using fully synthetic material.

## Files

- `synthetic_program_brief.md`: fictional redacted program brief.
- `synthetic_output_pack.md`: walkthrough of expected prepare-only outputs.

## Non-Negotiable Boundaries

This demo does not contain and must not be expanded to contain:

- real companies
- real domains
- real targets
- real accounts
- real vulnerabilities
- credentials
- tokens
- cookies
- session data
- personal data
- payment data
- classroom data
- payloads
- exploit steps
- scanning instructions
- target interaction instructions
- browser automation
- crawling
- automatic submission

## Demo Use

Allowed:

- explain the prepare-only product concept
- show expected artifact names
- explain blocker logic
- explain evidence-hygiene structure
- explain human-review gates

Not allowed:

- presenting the demo as a real finding
- presenting the demo as proof of target testing
- using it to justify live testing
- using it to imply guaranteed revenue, bounty payout, or accepted findings

## Review Requirement

Before external use, this demo should be reviewed for legal, privacy, tax, business, and messaging accuracy.
