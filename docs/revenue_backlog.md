# Safe Revenue Backlog

Status: draft

This backlog converts `bugos` into a revenue-ready product/service without allowing unsafe autonomous security activity.

## Phase 1: Productization

### REV-001: Define core offer

Goal: Package `bugos` as an offline-first prepare-only workflow for authorized security researchers and teams.

Deliverables:

- one-paragraph positioning
- included artifacts list
- excluded activities list
- buyer profile
- acceptance criteria

Safety gate:

- Must not imply scanning, exploitation, target interaction, credential testing, or automatic submission.

### REV-002: Create service menu

Goal: Define three safe services:

1. Redacted Program Brief Intake Pack
2. Evidence Hygiene and Report Readiness Review
3. Bug Bounty Operations QA Template Pack

Deliverables:

- service description
- scope
- boundaries
- buyer prerequisites
- output artifacts
- pricing draft

Safety gate:

- Must include `prepare-only`, `human-review required`, and `no automatic submission` language.

### REV-003: Draft landing page copy

Goal: Create copy for a simple landing page.

Sections:

- headline
- subheadline
- what it does
- what it does not do
- who it is for
- deliverables
- safety-first promise
- call to action

Safety gate:

- No income guarantees. No bounty guarantees. No autonomous hacking claim.

## Phase 2: Demo Assets

### REV-004: Build synthetic demo pack

Goal: Create a fully synthetic demo program brief and generated output screenshots/text snippets.

Deliverables:

- synthetic brief
- sample output pack
- README walkthrough

Safety gate:

- No real target, real domain, real company, real credential, or real vulnerability.

### REV-005: Create sample buyer handoff

Goal: Make the customer-facing handoff clearer.

Deliverables:

- delivery checklist
- redaction checklist
- human-review checklist
- final acceptance checklist

Safety gate:

- Customer must remain responsible for authorization, real testing, and submission.

## Phase 3: Trust and Risk Controls

### REV-006: Add data-handling notice

Goal: Explain how customers should redact briefs/evidence before using the workflow.

Deliverables:

- privacy notice draft
- data-minimization checklist
- prohibited input list

Safety gate:

- No secrets, tokens, cookies, personal data, payment data, classroom data, or third-party private data.

### REV-007: Add terms-of-use draft notes

Goal: Prepare non-lawyer notes for future legal review.

Deliverables:

- permitted-use summary
- prohibited-use summary
- no-guarantee statement
- human-responsibility statement

Safety gate:

- Must be reviewed by a qualified professional before external use.

## Phase 4: Sales Preparation

### REV-008: Create manual outreach draft

Goal: Draft a non-automated message for potential users.

Deliverables:

- short message
- longer message
- FAQ reply snippets

Safety gate:

- No automated outreach. No scraping. No spam. No income guarantees.

### REV-009: Create pricing hypothesis

Goal: Define conservative pricing options.

Options:

- Basic intake pack
- Standard review pack
- Template bundle
- Custom operations QA package

Safety gate:

- Pricing is a hypothesis only. No revenue guarantee.

## Phase 5: Codex Autonomy Loop

### REV-010: Add local no-change check

Goal: Codex should stop when no concrete safe revenue task exists.

Deliverables:

Safety gate:

- No arbitrary PRs. No broad housekeeping.
