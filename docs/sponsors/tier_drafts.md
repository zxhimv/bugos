# bugos Sponsor Tier Drafts

Status: draft for human review

These are safe GitHub Sponsors tier ideas. They are not active sponsorship tiers and should be reviewed before use in GitHub Sponsors.

## Tier Principles

- Keep benefits informational and non-operational.
- Do not promise security findings, bounties, payouts, target testing, scanning, exploitation, payloads, credentials, or report submission.
- Do not accept private customer data through sponsorship.
- Do not provide unsafe priority work.
- Keep all benefits compatible with the prepare-only boundary.

## Tier 1: Supporter

Purpose: general support for conservative offline-first tooling.

Possible wording:

`Support ongoing maintenance of bugos as a conservative, offline-first prepare-only workflow helper.`

Safe benefits:

- sponsor recognition if GitHub profile settings allow it
- public thanks where appropriate
- no operational service commitment

Do not include:

- custom research
- bug finding
- report review for real targets
- private support involving sensitive data

## Tier 2: Documentation Supporter

Purpose: support documentation, synthetic demos, and safe workflow templates.

Possible wording:

`Support documentation, synthetic examples, and human-review workflow templates for bugos.`

Safe benefits:

- periodic public progress notes
- sponsor-visible high-level roadmap notes, if GitHub supports them
- no private data processing

Do not include:

- customer-specific target review
- exploit or payload work
- guaranteed delivery dates unless separately reviewed

## Tier 3: Safety Supporter

Purpose: support guardrails, local checks, and regression tests.

Possible wording:

`Support safety checks, conservative defaults, and regression tests for bugos.`

Safe benefits:

- public acknowledgement
- sponsor-visible summaries of safety improvements, if GitHub supports them
- no operational security testing

Do not include:

- target testing
- scanner development
- vulnerability validation
- report submission assistance

## No-Go Tier Benefits

Do not offer:

- access to unsafe features
- exploit development
- payload packs
- scanning services
- credential testing
- target testing
- bounty hunting
- report submission
- revenue guarantees
- payout guarantees
- private data review through sponsorship

## Before Publishing Tiers

Review:

- GitHub Sponsors rules and profile settings
- tax and payout implications
- legal wording
- refund/cancellation expectations
- whether benefits create contractual obligations
