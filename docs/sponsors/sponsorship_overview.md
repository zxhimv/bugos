# bugos Sponsorship Overview

Status: draft for human review

This document explains safe sponsorship boundaries for `bugos`.

## Purpose

Sponsorship supports continued work on conservative, offline-first, human-reviewed security workflow tooling.

## What Sponsorship Supports

Sponsorship may support:

- maintenance of local prepare-only workflows
- documentation and synthetic demos
- safety checks and regression tests
- conservative report-readiness tooling
- human-review guardrail improvements
- privacy-preserving template work

## What Sponsorship Does Not Buy

Sponsorship does not buy:

- target testing
- scanning
- exploitation
- payload generation
- credential handling
- password-reset testing
- social engineering
- bug bounty submissions
- guaranteed findings
- guaranteed payouts
- guaranteed revenue
- priority access to unsafe capabilities
- custom work involving real targets or private data

## Sponsorship Boundary Statement

`bugos` is not a scanner, exploit framework, payload generator, credential-testing tool, target-interaction tool, browser-automation agent, crawler, or submission bot. Sponsorship does not change those boundaries.

## GitHub Sponsors Setup

Repository-level sponsor display is configured through `.github/FUNDING.yml`.

The GitHub Sponsors account, payout, tax, identity, billing, and sponsor-tier setup must be completed manually in GitHub by the account owner.

## Safe Sponsor Communication

Use language like:

`Sponsor conservative offline-first tooling for safer security workflow preparation.`

Do not use language like:

- sponsor me to find bugs
- sponsor me for guaranteed bounty results
- sponsor me for target testing
- sponsor me for exploit work
- sponsor me for report submission

## Review Requirement

Before enabling or promoting sponsorship publicly, review:

- GitHub Sponsors eligibility and account settings
- payout and tax details
- public profile wording
- sponsor-tier wording
- legal, privacy, tax, and platform obligations
